On building tools you actually want to use

tools software craft

There is a specific kind of frustration that comes from using software that was clearly designed for someone else. The menus are wrong. The keyboard shortcuts feel arbitrary. The thing you need is always one level deeper than it should be.

I have been thinking about this a lot lately, mostly because I keep building small tools for myself — a logging CLI, a local RSS reader, a script that emails me a summary of my notes every Sunday. None of these are impressive. All of them are deeply satisfying to use.

The grain of the tool

Every tool has a grain, like wood. You can work against it, but you will feel the resistance. The best tools have a grain that matches how you already think. You reach for them without deciding to.

Most commercial software optimizes for the median user. That is a reasonable business decision and a bad fit for any specific person. When you build your own tools, you are optimizing for an audience of one, and that changes everything about the design process.

The tools we use shape the thoughts we can have. Build tools that let you think thoughts you could not think otherwise.
